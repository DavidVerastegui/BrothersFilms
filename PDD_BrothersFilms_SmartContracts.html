Process Design Document (PDD)

BrothersFilms Smart Contracts System

Fecha: Febrero 2026 | Autor: Antigravity AI / David Verastegui | Versión: 1.0

1. Propósito y Alcance

Proceso: Generación, firma digital y administración de contratos de Fotografía y Video para Bodas y XV Años.

Descripción: Documentación de la automatización y transición de un flujo de contratos estáticos (Word/PDF local) a una arquitectura de Aplicación Web Descentralizada (SPA) alojada en GitHub Pages, con cálculos predictivos, enlaces estado-persistentes, telemetría y generación automática de PDFs firmados digitalmente.

2. Visión del Proceso (As-Is vs To-Be)

As-Is (Proceso Anterior) To-Be (Proceso Automatizado)
  • Interacción manual en Microsoft Word o PDF estático.
  • Cálculos de abonos y saldos restantes de forma manual propensos a errores.
  • Traspaso inseguro del documento por intermediarios.
  • Ausencia de validación digital interactiva y notificaciones sistematizadas.
  • Captura de datos en una Aplicación Web Inteligente (HTML/JS).
  • El sistema calcula y valida en tiempo real los totales y saldos, desplegando alertas ("✅ PAGADO").
  • Integración de Base de Datos en URL (parámetros `?`) para persistencia segura de la sesión.
  • El cliente plasma su Firma Digital en pantalla táctil/mouse.
  • El sistema inyecta datos al PDF y dispara correos por EmailJS automáticamente.

4. Secuencia Detallada de Pasos

Fase Acción del Sistema / Rol Descripción Técnica
1. Inicialización Administrador / UI Rellenado de campos semánticos y selección dinámica de planes (Básico, Standard, Premium/Rock Star) controlando reglas CSS de visibilidad (`display:flex/none`).
2. Motor Lógico Lógica JS (`updateSaldo`) Suma interactiva de los arreglos de inputs monetarios. Si el saldo remanente es $0, inyecta distintivos verdes y bloquea deudas adicionales.
3. Serialización Sistema (`generarLink`) Se captura toda la entrada del DOM, incluyendo la firma del administrador extraída del Canvas a un String Base64, para armar una cadena inyectada a `URLSearchParams`.
4. Modo Cliente Cliente / UI Al detectar bandera `?modo=cliente` el sistema congela toda edición (readonly) garantizando integridad de datos dictada por el administrador.
5. Consolidación Documental `html2canvas` + `jsPDF` Se realiza un renderizado vectorial-rasterizado (`scale: 2`) del contenido visible, mapeando coordinadas a un objeto binario PDF estructurado.
6. Notificación Mail API Externa (`EmailJS`) Consumo de Webhook seguro hacia el proveedor para orquestar la notificación de plantillas de correo sin necesidad de microservicios backend complejos.

5. Entorno Tecnológico y Dependencias

  • Frontend Core: HTML5 Semántico, CSS3 sin frameworks, JS Moderno (ES6+).
  • Dependencias de Software (CDN): `html2canvas v1.4.1`, `jsPDF v2.5.1`, `@emailjs/browser v4`.
  • Despliegue de Infraestructura: Entorno Estático Distribuido mediante GitHub Pages (SSL Automático).
  • Arquitectura Funcional: Serverless & Stateless (Data-en-URL).
